# Задачи для канбан-доски

Готовый список задач из плана реализации. Каждую задачу заведи как issue в
своём репозитории на GitHub, затем добавь на канбан-доску (см. раздел
«Канбан-доска в GitHub Projects» в гиде). Статусы указывают, в какую колонку
поместить задачу при создании доски.

> Как создать issue: на GitHub открой свой репозиторий → вкладка **Issues** →
> зелёная кнопка **New issue** → заголовок и описание из таблицы ниже →
> **Submit new issue**. Лейблы назначай справа в панели.

## Готово (Done)

Эти задачи уже выполнены — при создании доски положи их в колонку **Done**.

### 1. Выбрать версию YOLO
**Лейблы:** `documentation`, `research`
**Описание:** Изучить актуальные версии YOLO (YOLOv8, YOLOv9, YOLOv10, YOLO11),
сравнить баланс скорости и точности. Для старта выбран `yolov8s.pt` (small) —
хороший баланс; для embedded/CPU — `yolov8n.pt`.

### 2. Изучить теоретическую базу
**Лейблы:** `documentation`, `research`
**Описание:** Архитектура YOLO (однопроходный детектор, сетка, предсказание
bbox/conf/cls), bounding box и IoU, confidence score, Non-Maximum Suppression,
mAP@0.5 и mAP@0.5:0.95, transfer learning и fine-tuning.

### 3. Подготовить среду разработки
**Лейблы:** `setup`, `chore`
**Описание:** Создать виртуальное окружение (Python 3.9–3.11), установить
PyTorch (CPU или CUDA), ultralytics, opencv-python, matplotlib. Проверить
инференс `yolo predict model=yolov8s.pt source=0`.
